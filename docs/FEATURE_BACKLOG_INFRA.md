# NexusPurge 기능 개선 백로그
## 🚀 기능 백로그
### A. 안전장치 · 실수 방지 (Guardrails)
| 아이디어 |  설명 |
|---|---|
| 배포 전  Diff | 업로드 전 Local↔Remote 파일 목록을 나란히 비교하는 Diff 다이얼로그나 총 용량/건수 변화 계산은 없음 |

### C. CDN Purge 검증 · 쿼터/비용 관리
| 아이디어  | 설명 |
|---|---|
| Purge 중복 요청 방지 |  같은 경로를 짧은 시간에 여러 번 Purge해도 그대로 다 호출됨. 직전 Purge와 겹치면 합치거나 스킵 |
| Purge 후 캐시 워밍(프리페치) |  Purge만 하고 끝. 무효화 직후 주요 파일을 미리 요청해 캐시를 채우는 옵션 없음 |
| CDN별 요청 재시도/백오프 정책 커스터마이즈 |  공통 재시도 로직(`utils/retry.rs`)은 있으나, CDN마다 다른 재시도 전략을 프로필에서 설정하는 기능은 없음 |
| 파일 타입별 Cache-Control 템플릿 |  프로필당 기본 Cache-Control 하나(`defaultCacheControl`)만 있음. 확장자별(이미지 1년, HTML no-cache 등) 세분화 템플릿은 없음 |
| CDN별 SSL 인증서 만료 모니터링 |  커스텀 CDN 도메인의 인증서 만료를 미리 경고하는 기능 없음 |
| CDN 자동 장애조치(Failover) 안내 |  여러 CDN을 등록해도 각각 수동 선택/동시 Purge만 가능. 한 CDN이 무응답일 때 대체 CDN 안내 기능은 없음 |
| CDN 캐시 히트율/트래픽 통계 연동 |  CloudFront/Akamai 등 리포팅 API를 끌어와 히트율·트래픽을 보여주는 대시보드 없음 

### D. 감사(Audit) & 컴플라이언스 리포트
| 아이디어 | 설명 |
|---|---|
| Purge 이력 검색/필터 (날짜, provider, 성공/실패) |  LogPanel에서 최근 이력은 보여도 조건별 검색은 없음 |
| 활동 로그 대시보드 | 여러 사용자를 구분해 보여주는 대시보드는 없음(단일 사용자 데스크톱 앱 전제). A항목의 역할 게이팅이 완성되면 함께 구현 시 시너지 |

### E. 엔터프라이즈 네트워크 & 진단
| 아이디어 | 설명 |
|---|---|
| 네트워크 끊김 시 자동 재연결/큐 보존 | 전송 중 네트워크가 끊기면 실패 처리만 됨 — 재접속 후 이어서 진행하는 로직 없음 |


### G. S3 운영 편의성
| 아이디어 |  설명 |
|---|---|
| S3 메타데이터 & HTTP 헤더 일괄 편집기 |  이미 저장된 수천 개 객체의 `Cache-Control`/`Content-Type`/`Content-Encoding` 등을 재업로드 없이 일괄 변경하는 기능 없음 |
| S3 깊은 경로 즐겨찾기 / 북마크 |  5단계 이상 깊은 S3 구조를 매번 탐색해야 함. 자주 가는 경로를 저장해두고 바로 이동하는 기능 없음 |
| 전송 개별 일시정지/재개 |  전체 취소(`cancel_transfer`)는 있지만, 진행 중인 특정 파일만 멈췄다 이어받는 기능은 없음 |
| 패널 내 검색/필터 |  현재 목록에서 파일명으로 즉시 필터링하는 입력창 없음(Profile 목록 검색은 있음) |
| 압축 후 업로드 (zip) |  여러 파일을 압축해서 하나로 올리는 옵션 없음 |
| 다단계 패널 (Local → Stg → Prd) |  지금은 "로컬 + 원격(S3) 1개" 고정 구조. 여러 원격 패널을 동시에 띄우거나 단계별 승격 플로우를 만들려면 패널 레이아웃과 전송 아키텍처 자체를 다시 설계해야 함 |
| S3 ↔ S3 (프로필 간) 전송/승격 |  CopyObject는 같은 버킷 내 이름변경(rename)에만 쓰임. 버킷/계정이 다른 두 프로필 간 복사 기능은 백엔드에 아예 없음 — Dev→Staging→Prod 승격 플로우의 기반 기능 |
| 양방향(mirror) 동기화 |  지금은 로컬→S3 단방향 Smart Sync만 있음. S3가 변경되면 로컬에 반영하는 모드는 없음 |

### H. 일반 UX / 접근성 (인프라 우선순위는 낮으나 백로그 보존)
| 아이디어 | 설명 |
|---|---|---|
| 다국어 (i18n) | 프론트 전체 + Rust 백엔드 에러 메시지까지 한글 하드코딩. 전수 텍스트 추출 작업 필요 |
| 접근성 (스크린 리더, 고대비, 키보드 전용 조작) | 별도 접근성 감사 필요 |
| 파일 미리보기 (이미지/텍스트) | 업로드/다운로드 전에 내용을 바로 확인하는 기능 없음 |
| 명령 팔레트 (Ctrl+K 스타일) | 자주 쓰는 동작을 검색해서 실행하는 UI 없음 |
| 자동 업데이트 | Tauri updater 플러그인 미설치로 보임(확인 필요) |
