# UploadTool 요구사항

## 개요

**UploadTool의 요구사항은 AWS S3 파일 관리, CDN Purge 자동화, Profile 관리, 인증 연동 준비, 로그/결과 처리 구조**로 구분됩니다.

본 문서는 기존 CDN 업로드/Purge 툴을 대체하기 위해 필요한 기능 요구사항을 정리한 것입니다.

---

## 동일 파일 판단 기준

| 항목 | 기준 |
| --- | --- |
| 기본 판단 방식 | MD5/ETag 비교 |

## S3 Key와 CDN URL 매핑 요구사항

S3 Key와 CDN URL 매핑은 UploadTool의 핵심 로직 중 하나입니다.

| 항목 | 내용 |
| --- | --- |
| S3 Key 예시 | `prod/assets/app.js` |
| CDN URL 예시 | `https://cdn.example.com/prod/assets/app.js` |
| 처리 방식 | 공통 함수로 S3 Key 또는 Object Path를 CDN URL로 변환 |
| 기본 규칙 | CDN Domain의 trailing slash 제거 |
| 기본 규칙 | Object Path의 leading slash 제거 |
| 결과 형태 | `domain/path` 형태로 조합 |
| CDN별 Prefix 차이 | 고객사 확인 후 반영 |
| Query String 포함 여부 | 고객사 확인 후 반영 |

## 정리

> UploadTool 요구사항은 **AWS S3 파일 관리 기능**, **CDN Purge 자동화**, **S3 Key ↔ CDN URL 매핑**, **Profile 관리**, **외부 인증 모듈 연동 준비**, **작업 로그/결과 처리 구조**로 구성됩니다.
> 
> 
> 1차 개발은 기존 툴 기능 재현을 목표로 하며, LG U+ CDN과 효성 CDN은 API 문서 수령 전까지 Stub Adapter로 구성합니다.
