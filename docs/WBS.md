# NexusPurge (CDN Upload Tool) — WBS (작업분류체계)

**문서 목적**: 프로젝트 착수부터 현재까지의 개발 범위를 작업 단위로 분류하여 고객사에 진행 현황을 보고합니다.
**기준일**: 2026-07-14
**대상 시스템**: NexusPurge — S3 업로드 · 다중 CDN 자동 Purge Windows 데스크톱 도구
**개발 기간**: 2026-04-23 ~ 2026-07-14 (약 12주, 커밋 84건)

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 제품명 | NexusPurge (CDN Upload Tool) |
| 형태 | Windows 데스크톱 애플리케이션 (설치 파일 `.msi` 배포) |
| 핵심 기능 | FTP 스타일 듀얼 패널 S3 업로드 + 파일 덮어쓰기 감지 시 CDN 캐시 자동 무효화(Purge) |
| 지원 CDN | AWS CloudFront, Akamai, LG U+ CDN, KT CDN, 효성 ITX (5종, 멀티 CDN 동시 Purge 지원) |
| 기술 스택 | Frontend: React 18 + TypeScript + Vite + Zustand · Backend: Rust + Tauri 2 + Tokio · AWS SDK for S3 |
| 자격증명 보관 | OS 자격 증명 관리자(Windows Credential Manager) 암호화 저장 |
| 현재 규모 | Rust 소스 34개 파일 · Tauri 커맨드 43개 · 단위 테스트 61개(Rust 31 + Frontend 30) 전체 통과 |
