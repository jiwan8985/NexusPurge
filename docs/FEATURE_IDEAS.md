# NexusPurge 기능 개선 아이디어 백로그

---

## 1. 워크플로우 / 패널 구조

| 아이디어 | 현재 상태 | 설명 |
|---|---|---|
| 다단계 패널 (Local → Stg → Prd) | ❌ 없음 | 지금은 "로컬 + 원격(S3) 1개" 고정 구조. 여러 원격 패널을 동시에 띄우거나 단계별 승격 플로우를 만들려면 패널 레이아웃과 전송 아키텍처 자체를 다시 설계해야 함 |
| S3 ↔ S3 (프로필 간) 전송/승격 | ❌ 없음 | CopyObject는 같은 버킷 내 이름변경(rename)에만 쓰임. 버킷/계정이 다른 두 프로필 간 복사 기능은 백엔드에 아예 없음 |
| 즐겨찾기 / 북마크 (자주 쓰는 S3 경로) | ❌ 없음 | 매번 폴더를 탐색해야 함. 자주 가는 경로를 저장해두고 바로 이동하는 기능 |
| 패널 내 검색/필터 | ❌ 없음 | 현재 목록에서 파일명으로 즉시 필터링하는 입력창 없음 (Profile 목록 검색은 있음) |
| 양방향(mirror) 동기화 | ❌ 없음 | 지금은 로컬→S3 단방향 Smart Sync만 있음. 반대로 S3가 변경되면 로컬에 반영하는 모드는 없음 |


## 2. 언어 / 접근성

| 아이디어 | 현재 상태 | 설명 |
|---|---|---|
| 다국어 (i18n) | ❌ 없음 | 프론트 전체 + Rust 백엔드 에러 메시지까지 한글 하드코딩. 전수 텍스트 추출 작업 필요 |
| 접근성 (스크린 리더, 고대비, 키보드 전용 조작) | ❌ 없음 | 별도 접근성 감사 필요 |
| 커스텀 단축키 | ⚠️ 부분적 | 일부 키 이벤트(ESC로 메뉴 닫기 등)만 있고, 사용자가 직접 바인딩을 바꾸는 기능은 없음 |

## 3. 전송 / 큐 관리

| 아이디어 | 현재 상태 | 설명 |
|---|---|---|
| 전송 개별 일시정지/재개 | ⚠️ 부분적 | 전체 취소(`cancel_transfer`)는 있지만, 진행 중인 특정 파일만 멈췄다 이어받는 기능은 없음 |
| 대역폭 제한 | ❌ 없음 | 동시 전송 개수 제한(`maxConcurrentTransfers`)은 있지만 속도(KB/s) 제한은 없음 |
| 실패 자동 재시도 정책 세분화 | ⚠️ 부분적 | 재시도 로직(`utils/retry.rs`)은 있으나, 사용자가 "몇 번, 몇 초 간격" 등을 세밀하게 조정하는 UI는 없음 |
| 예약 업로드/동기화 (스케줄링) | ❌ 없음 | 특정 시간에 자동으로 Smart Sync를 실행하는 기능 없음 |
| 대용량 폴더 업로드 시 사전 용량/건수 미리보기 | ⚠️ 부분적 | Purge 배치 정책(경고/비권장 임계값)은 있지만 업로드 자체의 사전 용량 계산은 없음 |

## 4. CDN / Purge

| 아이디어 | 현재 상태 | 설명 |
|---|---|---|
| Purge 결과 리포트 내보내기 (CSV) | ❌ 없음 | `operation_log.rs`에 "CSV export는 리포트 컬럼 확정 후 추가" TODO만 있음 |
| Purge 영향 범위 시각화(미리보기) | ❌ 없음 | 지금은 대상 경로 텍스트 목록만 보여줌 |
| Purge 후 캐시 워밍(프리페치) | ❌ 없음 | Purge만 하고 끝. 무효화 직후 주요 파일을 미리 한 번 요청해서 캐시를 채워두는 옵션 |
| CDN별 요금/쿼터 안내 | ⚠️ 부분적 | 배치 크기 경고/비권장 임계값은 있지만, 실제 CDN 과금 체계 안내는 없음 |
| Purge 이력 검색/필터 (날짜, provider, 성공/실패) | ❌ 없음 | LogPanel에서 최근 이력은 보여도 조건별 검색은 없음 |

## 6. 협업 / 멀티유저

| 아이디어 | 현재 상태 | 설명 |
|---|---|---|
| 활동 로그 대시보드 (누가 언제 무엇을) | ❌ 없음 | 현재 로그는 "무슨 일이 있었는지"는 상세히 남기지만, 여러 사용자를 구분해 보여주는 대시보드는 없음(단일 사용자 데스크톱 앱 전제) |

## 7. 안전장치 / 감사

| 아이디어 | 현재 상태 | 설명 |
|---|---|---|
| 감사 로그 내보내기 (CSV/PDF, 컴플라이언스용) | ❌ 없음 | `audit-*.log`는 텍스트 파일로만 존재 |
| S3 버전 관리(Versioning) 조회/복원 UI | ❌ 없음 | Properties 다이얼로그에서 버전 ID는 보여주지만, 이전 버전 목록 조회·복원 기능은 없음 |
| 삭제 되돌리기(휴지통) | ❌ 없음 | 삭제 시 확인창은 있지만 복구 기능은 없음(S3 Versioning UI와 연계 가능) |
| 다중 인스턴스 동시 쓰기 충돌 방지 | ❌ 없음 | 위 1번 항목과 연결 — profiles.json 쓰기 시 파일 잠금 필요 |

## 8. 편의 기능

| 아이디어 | 현재 상태 | 설명 |
|---|---|---|
| 파일 미리보기 (이미지/텍스트) | ❌ 없음 | 업로드/다운로드 전에 내용을 바로 확인하는 기능 없음 |
| 명령 팔레트 (Ctrl+K 스타일) | ❌ 없음 | 자주 쓰는 동작을 검색해서 실행하는 UI 없음 |
| 자동 업데이트 | ❌ 없음 | 확인 필요 — Tauri updater 플러그인 미설치로 보임 |
| 압축 후 업로드 (zip) | ❌ 없음 | 여러 파일을 압축해서 하나로 올리는 옵션 없음 |

## 10. CDN 관점

| 아이디어 | 현재 상태 | 설명 |
|---|---|---|
| CDN 캐시 히트율/트래픽 통계 연동 | ❌ 없음 | CloudFront/Akamai 등이 제공하는 리포팅 API를 끌어와 히트율·트래픽을 보여주는 대시보드 없음 — 지금은 Purge 성공/실패만 봄 |
| Purge 중복 요청 방지(디바운스) | ❌ 없음 | 같은 경로를 짧은 시간 안에 여러 번 Purge 요청해도 그대로 다 호출됨 — 직전 Purge와 겹치면 합치거나 스킵하는 로직 없음 |
| CDN別 SSL 인증서 만료 모니터링 | ❌ 없음 | 커스텀 CDN 도메인의 인증서 만료를 미리 경고해주는 기능 없음 |
| CDN 자동 장애조치(Failover) | ❌ 없음 | 여러 CDN을 등록해도 각각 수동 선택/동시 Purge만 가능 — 한 CDN이 응답 없을 때 자동으로 다른 CDN으로 트래픽 안내(예: 고객사에 DNS 전환 안내)하는 기능은 없음 |
| CDN Provider 추가 확장 (Fastly, Cloudflare, Azure CDN, GCP Cloud CDN) | ❌ 없음 | 현재 5종(CloudFront/Akamai/LG U+/KT/효성ITX)만 지원. 어댑터 구조(`CdnAdapter` trait)는 확장 가능하게 되어 있어 추가 자체는 기존 패턴을 따르면 됨 |
| Purge 후 자동 검증(엣지 응답 재확인) | ⚠️ 부분적 | 우클릭 속성에서 수동으로 "실시간 확인"은 가능하지만, Purge 완료 직후 자동으로 엣지 응답을 재확인해 "진짜로 갱신됐는지" 검증하는 자동화는 없음 |
| CDN별 요청 재시도/백오프 정책 커스터마이즈 | ⚠️ 부분적 | 공통 재시도 로직(`utils/retry.rs`)은 있지만 CDN마다(예: 효성ITX는 대용량에서 타임아웃 이슈) 다른 재시도 전략을 프로필에서 설정하는 기능은 없음 |
| 파일 타입별 Cache-Control 템플릿 | ⚠️ 부분적 | 프로필당 기본 Cache-Control 하나(`defaultCacheControl`)만 있음 — 확장자별(이미지는 1년, HTML은 no-cache 등) 세분화된 템플릿은 없음 |
