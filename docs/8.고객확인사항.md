# 고객 확인사항

본 문서는 API 문서 또는 고객 운영 기준 수령 전까지 확정 구현하지 않을 항목을 관리합니다.

## CDN/API

| 구분 | 확인사항 |
| --- | --- |
| LG U+ CDN | Purge API 문서, 인증 방식, 요청 Payload, Rate Limit |
| 효성 CDN | Purge API 문서, 인증 방식, 요청 Payload, Rate Limit |
| KT CDN | Purge API 문서, 인증 방식, 요청 Payload, Rate Limit |
| Akamai | URL Purge 유지 여부 또는 CP Code Purge 필요 여부 |
| CDN 공통 | 1회 Purge URL 제한, 요청 간격, 완료 상태 Polling 필요 여부 |

## Purge/업로드 정책

| 구분 | 확인사항 |
| --- | --- |
| Batch Size | 기본 1,000개 기준 적정성 |
| 대용량 경고 | 5,000개 이상 경고 기준 적정성 |
| 미권고 경고 | 10,000개 이상 미권고 기준과 승인 절차 |
| Purge 범위 | 전체/개별/일부 Purge 운영 방식 |
| Overwrite/Skip | 정책별 Purge 대상 산정 방식 |
| 삭제 파일 | 삭제 파일 Purge 필요 여부 |
| 와일드카드 | CDN별 와일드카드 Purge 지원/사용 여부 |
| Query String | Query String 포함 Purge 필요 여부 |

## Metadata/Header

| 구분 | 확인사항 |
| --- | --- |
| Content-Type | 자동 적용 규칙 |
| Cache-Control | 기본값 및 경로별 예외 규칙 |
| 사용자 Header | 허용 Header 목록과 보안 제한 |
| 사용자 Metadata | 허용 Metadata 규칙 |
| 실패 재시도 | Metadata/Header 적용 실패 시 수동 재시도 절차 |

## Profile/인증

| 구분 | 확인사항 |
| --- | --- |
| Profile 배포 | 암호화 Profile Import/Remove 파일 형식 |
| 권한 | 일반 사용자와 관리자 권한 분리 |
| 범위 | 프로젝트 단위 Profile과 사용자 단위 Profile 분리 기준 |
| 다중 CDN | 하나의 Profile 내 복수 CDN Provider 운영 필요 여부 |
| AI LB | 인증키/세션 연동 방식과 만료/갱신 절차 |
| 외부 인증 | 자체 로그인/계정 DB 없이 연동할 인증 시스템 문서 |

## 로그 적재

| 구분 | 확인사항 |
| --- | --- |
| 대상 Bucket | 고객 제공 S3 로그 Bucket |
| Prefix | 환경/프로젝트/사용자별 Prefix 규칙 |
| 포맷 | JSON 필드와 보관 기간 |
| 대상 로그 | 업로드/다운로드/Purge/오류/Metadata 실패 포함 여부 |
| 재시도 | 적재 실패 재시도 횟수, 간격, 최종 실패 처리 |

## 설치/배포/패치

| 구분 | 확인사항 |
| --- | --- |
| 테스트 버전 | 설치형 패키지 기준 확인 |
| Windows Server | Windows Server 2025/2022/2019 지원 가능 여부 |
| Desktop OS | Windows/macOS/Linux 최근 3개년 지원 범위 |
| Unix 서버 | Unix 서버 지원 가능 여부 |
| 자동 패치 | 자동 패치/보안 업데이트 구조 |
| 무재시작 패치 | 재시작 없는 패치 가능 여부 검토, 확정 금지 |
