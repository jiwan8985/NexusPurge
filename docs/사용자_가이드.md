## 1. 시스템 요구사항

| 항목 | 요구사항 |
|------|----------|
| **운영체제** | Windows 10/11 (64비트) |
| **인터넷 연결** | S3 버킷 및 CDN 접속을 위한 인터넷 연결 필요 |
| **권한** | 일반 사용자 권한으로 실행 가능 (관리자 권한 불필요) |
| **방화벽** | 아웃바운드 HTTPS(443) 허용 필요 |

## 2. 설치

### Windows (.msi 설치파일)

1. 제공받은 `NexusPurge_x.x.x_x64_en-US.msi` 파일을 실행합니다.
2. 설치 마법사의 안내에 따라 **다음(Next)** 버튼을 클릭합니다.
3. 설치가 완료되면 바탕화면 또는 시작 메뉴에서 **NexusPurge** 아이콘으로 실행합니다.

### Windows (.exe 설치파일)

1. 제공받은 `NexusPurge_x.x.x_x64-setup.exe` 파일을 실행합니다.
2. 설치 완료 후 바탕화면 단축아이콘이 생성됩니다.

> **참고** 설치 경로는 기본값(`C:\Program Files\NexusPurge`)을 그대로 사용하길 권장합니다.

## 3. 첫 실행 및 프로필 설정

NexusPurge를 처음 실행하면 연결된 프로필이 없어 빈 화면이 표시됩니다.  
S3 버킷에 접속하려면 **프로필**을 먼저 등록해야 합니다.

### 방법 A — 프로필 파일 가져오기 (권장)

담당자로부터 `.nxp` 또는 `.json` 형태의 프로필 파일을 제공받은 경우 아래 순서로 가져옵니다.

1. 상단 타이틀바에서 프로필 영역(좌측 상단)을 클릭합니다.
2. **프로필 관리** 창이 열리면 **가져오기(Import)** 버튼을 클릭합니다.
3. 제공받은 프로필 파일을 선택합니다.
4. 가져오기 완료 후 목록에 프로필이 나타납니다.
5. 프로필을 선택하고 **연결** 버튼을 클릭합니다.

### 방법 B — 직접 입력

S3 자격증명을 직접 입력하여 프로필을 생성합니다.

1. **프로필 관리** 창 → **새 프로필** 버튼을 클릭합니다.
2. 아래 항목을 입력합니다.

| 항목 | 필수 | 예시 | 설명 |
|------|:----:|------|------|
| 프로필 이름 | ✓ | `운영 배포` | 식별용 이름 (자유 입력) |
| 버킷 이름 | ✓ | `my-cdn-bucket` | AWS S3 버킷 이름 |
| 리전 | ✓ | `ap-northeast-2` | 버킷이 있는 AWS 리전 |
| Access Key ID | ✓ | `AKIA...` | IAM 액세스 키 ID |
| Secret Access Key | ✓ | `wJalrX...` | IAM 시크릿 액세스 키 |
| CDN 제공자 | | `AWS CloudFront` | CDN Purge를 사용하는 경우 선택 |
| Distribution ID | | `E1A2B3C4D5E6F7` | CloudFront Distribution ID |
| CDN 도메인 | | `cdn.example.com` | CDN 캐시 확인용 도메인 |

3. **저장** 버튼을 클릭합니다.
4. 저장된 프로필을 선택 후 **연결**을 클릭합니다.

> **보안** Secret Access Key는 Windows 자격증명 관리자에 암호화되어 저장됩니다.  
> 파일로 내보내거나 다른 사람에게 복사되지 않습니다.

## 5. 파일 탐색

### 로컬 패널 (왼쪽)

- **경로 클릭**: 현재 경로를 직접 클릭하거나 수정하여 이동합니다.
- **폴더 더블클릭**: 해당 폴더로 이동합니다.
- **`..` (상위 폴더)**: 목록 상단의 `..`을 더블클릭하면 상위 폴더로 이동합니다.
- **정렬**: 열 헤더(이름/크기/수정일)를 클릭하면 정렬 방향이 바뀝니다.

### S3 패널 (오른쪽)

- S3는 폴더처럼 보이지만 실제로는 `/`가 포함된 키(경로)입니다.
- 폴더 더블클릭 또는 경로 직접 입력으로 이동할 수 있습니다.

### 파일/폴더 선택

- **단일 선택**: 항목을 클릭합니다.
- **다중 선택**: `Ctrl` 키를 누른 채로 클릭합니다.
- **범위 선택**: `Shift` 키를 누른 채로 클릭합니다.

### 우클릭 메뉴

파일이나 폴더를 우클릭하면 아래 메뉴가 나타납니다.

| 메뉴 | 설명 |
|------|------|
| 업로드 | 선택 항목을 S3로 업로드 |
| 다운로드 | 선택 항목을 로컬로 다운로드 |
| 삭제 | 선택 항목 삭제 |
| CDN Purge | 선택 항목의 CDN 캐시 무효화 |
| Presigned URL 복사 | 임시 공개 URL 생성 및 클립보드 복사 |
| 이름 바꾸기 | 파일/폴더 이름 변경 |

## 6. 업로드

### 기본 업로드

1. **로컬 패널**에서 업로드할 파일 또는 폴더를 선택합니다.
2. 중앙의 `▶` 버튼을 클릭합니다.  
   (또는 파일을 드래그하여 S3 패널에 놓아도 됩니다.)
3. **동기화 미리보기** 창이 열려 변경 내역을 확인할 수 있습니다.  
   → [9. 동기화 미리보기](#9-동기화-미리보기) 참고
4. **업로드 시작** 버튼을 클릭합니다.

### Smart Sync (변경된 파일만 전송)

NexusPurge는 업로드 전 파일을 자동으로 비교합니다.

| 결과 | 의미 |
|------|------|
| **신규** | S3에 없는 파일 → 업로드 |
| **수정** | S3 파일과 내용이 다름 → 덮어쓰기 후 CDN Purge |
| **변경없음** | S3 파일과 동일 → **자동 스킵** (재전송 안 함) |

> 이미 배포된 파일과 동일한 내용이면 전송하지 않으므로 시간과 비용을 절약합니다.

### 업로드 진행 확인

업로드가 시작되면 우하단 또는 전송큐 탭에서 진행 상황을 확인할 수 있습니다.

- 파일별 진행률(%) 및 전송 속도(MB/s) 표시
- 전송 중 **취소** 버튼으로 중단 가능
- 완료 후 CDN Purge 결과가 함께 표시됩니다.

## 7. 다운로드

1. **S3 패널**에서 다운로드할 파일 또는 폴더를 선택합니다.
2. 중앙의 `◀` 버튼을 클릭합니다.
3. 로컬 패널에 표시된 현재 경로로 파일이 저장됩니다.

> 이미 동일한 파일이 로컬에 있어도 덮어쓰기를 진행합니다.  
> 중요한 파일은 미리 백업 후 다운로드하세요.

## 8. CDN Purge

CDN Purge는 배포한 파일이 CDN 캐시에 남아 있어 사용자에게 구 버전이 보이는 문제를 해결합니다.

### 업로드 시 자동 Purge

파일을 덮어쓰기하면(기존 파일 교체) **업로드 완료 직후 자동으로 CDN Purge**가 실행됩니다.  
작업 로그에서 `CDN Purge 완료` 메시지로 확인할 수 있습니다.

### 수동 Purge

특정 파일의 CDN 캐시를 직접 무효화하려면 아래 방법을 사용합니다.

**방법 1 — S3 패널에서 우클릭**

1. S3 패널에서 Purge할 파일을 선택합니다.
2. 우클릭 → **CDN Purge** 메뉴를 선택합니다.

**방법 2 — 툴바 Purge 버튼**

1. S3 패널에서 Purge할 파일을 선택합니다.
2. 툴바의 **CDN Purge** 버튼을 클릭합니다.
3. **CDN Purge 확인** 창에서 대상 경로를 확인합니다.
4. **Purge 실행** 버튼을 클릭합니다.

### Purge 확인 창 안내

| 메시지 | 설명 |
|--------|------|
| 일반 안내 | 선택한 경로 목록 및 개수 표시 |
| ⚠️ 대용량 경고 | 경로가 많을 경우 경고 표시 — 확인 후 진행 |
| ⚠️ 루트 전체 Purge | CDN 전체 캐시가 무효화됨 — 운영 중 사용 주의 |

> **루트 전체 Purge**는 모든 사용자에게 즉시 영향을 미칩니다.  
> CDN 원본 트래픽이 급증할 수 있으므로 신중히 실행하세요.

### Purge 결과 확인

Purge 완료 후 결과 창이 자동으로 열립니다.

- 성공: 무효화 요청 ID (Invalidation ID) 표시
- 실패: 실패 원인 및 재시도 안내

로그 패널의 **Purge이력** 탭에서도 과거 Purge 기록을 확인할 수 있습니다.

## 9. 동기화 미리보기

업로드 전 변경 내역을 먼저 확인하는 기능입니다.

### 미리보기 탭 설명

| 탭 | 설명 |
|----|------|
| **신규** | 로컬에만 있고 S3에 없는 파일 (업로드 예정) |
| **수정** | 양쪽 모두 있으나 내용이 다른 파일 (덮어쓰기 후 CDN Purge 예정) |
| **삭제** | S3에만 있고 로컬에 없는 파일 (이번 업로드에서 처리 안 됨) |
| **변경없음** | 양쪽이 동일한 파일 (스킵) |

### 진행 방법

1. 미리보기에서 각 탭을 확인합니다.
2. 예상치 못한 파일이 포함된 경우 **취소**를 클릭하고 로컬 파일을 확인합니다.
3. 이상이 없으면 **업로드 시작** 버튼을 클릭합니다.

## 10. Presigned URL 생성

S3 파일에 대한 임시 공개 URL을 생성합니다. 비공개 버킷의 파일을 특정 기간 동안만 공유할 때 사용합니다.

1. S3 패널에서 파일을 선택합니다.
2. 우클릭 → **Presigned URL 복사**를 선택합니다.
3. 유효 기간을 선택합니다.

| 유효 기간 | 사용 예 |
|-----------|---------|
| 15분 | 즉시 공유, 단기 테스트 |
| 1시간 | 일반 파일 공유 |
| 24시간 | 장기 공유, 다운로드 링크 제공 |

4. 클립보드에 복사된 URL을 공유합니다.

> URL이 만료되면 접근이 차단됩니다. 만료 전에 미리 재생성하세요.

## 11. 작업 로그 확인

화면 하단의 로그 패널에서 모든 작업 이력을 확인합니다.

### 탭 설명

| 탭 | 내용 |
|----|------|
| **작업로그** | 업로드·다운로드·삭제·CDN Purge 결과 실시간 표시 |
| **전송큐** | 현재 진행 중이거나 대기 중인 전송 목록 |
| **Purge이력** | 과거 CDN Purge 실행 내역 |

### 로그 상태 아이콘

| 표시 | 의미 |
|------|------|
| `OK` (초록) | 성공 |
| `WARN` (노랑) | 경고 (진행은 됨) |
| `ERR` (빨강) | 오류 (실패) |
| `INFO` (회색) | 일반 정보 |

## 12. 문제 해결

### 연결 오류

**증상**: 프로필 연결 시 "연결 실패" 또는 오류 메시지

| 원인 | 해결 방법 |
|------|----------|
| Access Key 오류 | Access Key ID / Secret Access Key를 다시 확인합니다. |
| 버킷 이름 오류 | 대소문자 포함 버킷 이름을 정확히 입력합니다. |
| 리전 오류 | 버킷이 속한 리전을 정확히 선택합니다. |
| IAM 권한 부족 | S3 읽기·쓰기 권한이 있는지 담당자에게 확인합니다. |
| 방화벽/프록시 | HTTPS(443) 아웃바운드 접근이 허용되어 있는지 확인합니다. |

### 업로드 오류

**증상**: 업로드 중 특정 파일에서 오류 발생

| 원인 | 해결 방법 |
|------|----------|
| 파일이 다른 프로그램에서 사용 중 | 해당 프로그램을 닫고 다시 시도합니다. |
| 파일명에 특수문자 포함 | 파일명의 특수문자를 제거하거나 영문으로 변경합니다. |
| 네트워크 불안정 | 인터넷 연결을 확인 후 재시도합니다. |
| S3 용량 한도 초과 | S3 버킷 용량을 확인합니다. |

### CDN Purge 오류

**증상**: Purge 실행 후 "실패" 표시

| 원인 | 해결 방법 |
|------|----------|
| CDN 자격증명 오류 | 프로필의 CDN 설정(Distribution ID, 도메인)을 확인합니다. |
| Distribution ID 오류 | CloudFront 콘솔에서 올바른 Distribution ID를 확인합니다. |
| IAM CloudFront 권한 없음 | `cloudfront:CreateInvalidation` 권한이 필요합니다. |
| 경로 수 초과 | 한 번에 너무 많은 경로를 Purge하는 경우 분할 실행합니다. |

### 파일이 업로드됐는데 CDN에서 구 버전이 보임

CDN 캐시가 남아 있는 경우입니다.

1. S3 패널에서 해당 파일을 선택합니다.
2. 우클릭 → **CDN Purge** 를 실행합니다.
3. 수분 내로 CDN 캐시가 갱신됩니다. (CDN에 따라 최대 10~15분 소요)

### 자동 Purge가 실행되지 않음

- 프로필에 CDN 제공자와 Distribution ID가 설정되어 있는지 확인합니다.
- 로그 패널 → **작업로그** 탭에서 오류 메시지를 확인합니다.

## 13. 자주 묻는 질문

**Q. 파일을 업로드했는데 웹사이트에 반영이 되지 않습니다.**  
A. 업로드 로그에서 CDN Purge 성공 여부를 확인하세요. Purge가 실행되지 않았다면 수동으로 CDN Purge를 실행합니다. Purge 후에도 수 분이 걸릴 수 있습니다.

---

**Q. 이미 S3에 있는 파일을 다시 업로드하면 어떻게 됩니까?**  
A. 파일 내용이 동일하면 자동으로 스킵(전송 안 함)합니다. 내용이 다르면 덮어쓰기 후 CDN Purge가 자동 실행됩니다.

---

**Q. 업로드를 실수로 시작했습니다. 취소할 수 있습니까?**  
A. 전송 진행 중 **취소** 버튼을 클릭하면 즉시 중단됩니다. 이미 전송 완료된 파일은 취소되지 않습니다.

---

**Q. 여러 버킷을 사용합니다. 어떻게 전환합니까?**  
A. 타이틀바의 프로필 영역을 클릭하면 프로필 목록이 나타납니다. 원하는 프로필을 선택하고 **연결** 버튼을 클릭하면 해당 버킷으로 전환됩니다.

---

**Q. Secret Access Key를 잊어버렸습니다.**  
A. Secret Access Key는 최초 입력 시 OS 보안 저장소에 저장되며, 화면에 다시 표시되지 않습니다. 분실 시 AWS IAM 콘솔에서 새 키를 발급 후 프로필을 다시 설정하세요.

---

**Q. Purge 요청 후 얼마나 기다려야 합니까?**  
A. CDN 제공자에 따라 다릅니다.

| CDN | 예상 소요 시간 |
|-----|--------------|
| AWS CloudFront | 1~3분 |
| Akamai | 수초~수분 |
| LG U+ CDN | 수분 |
| KT CDN | 수분 |

---

**Q. 로그를 파일로 저장할 수 있습니까?**  
A. 현재 버전에서는 UI 로그 패널에서 확인할 수 있으며, 파일 내보내기는 추후 지원 예정입니다.

---

**Q. 앱을 업데이트하면 프로필 정보가 사라집니까?**  
A. 프로필 정보는 앱과 별도 위치에 저장되므로 업데이트 후에도 유지됩니다.
