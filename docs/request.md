# Upload Tool 요구사항 통합 정리

## 1. 프로젝트 방향

Upload Tool은 기존 업로드/Purge 툴을 대체하는 신규 도구로, 1차 개발은 **업로드와 Purge 중심**으로 진행한다.

기존 기능 중 비교, Prefetch 등은 포함 가능하나 사용 빈도가 낮으므로 우선순위는 낮게 둔다.

주요 목적은 다음과 같다.

- Drag & Drop 기반 업로드
- 다운로드
- 전체/개별 Purge
- Overwrite / Skip 처리
- Metadata/Header 수동 입력
- 업로드/다운로드/Purge 로그 생성
- 고객 제공 Bucket으로 로그 적재
- 암호화 프로필 기반 사용 권한 제어
- AI LB 인증/프로필 시스템과 연동 가능한 구조 확보

---

## 2. 핵심 기능 요구사항

### 2.1 업로드

업로드는 Drag & Drop 방식을 기본으로 지원한다.

주요 기능은 다음과 같다.

- Drag & Drop 업로드
- 대용량 업로드
- 업로드 전/후 파일 비교
- Overwrite / Skip 선택
- 업로드 시 Metadata/Header 입력 및 수정
- 업로드 시 자동 Metadata 업로드
- 자동 Metadata 업로드 실패 시 수동 업로드 가능
- 업로드 로그 생성
- 고객 제공 Bucket으로 로그 적재

업로드 시 Metadata/Header는 사용자가 수동으로 입력하거나 정정할 수 있어야 한다.

---

### 2.2 다운로드

다운로드 기능을 지원한다.

주요 기능은 다음과 같다.

- 프로필 기반 다운로드
- 다운로드 로그 생성
- 고객 제공 Bucket으로 로그 적재

---

### 2.3 Purge

Purge는 업로드와 함께 핵심 기능으로 개발한다.

주요 기능은 다음과 같다.

- 전체 Purge
- 개별 Purge
- 일부 Purge
- 통합 Purge
- 대용량 Purge
- Purge Overwrite
- Purge Skip
- 자동 Purge 여부 체크 버튼
- 자동 Purge 실행 버튼
- Purge 로그 생성
- 고객 제공 Bucket으로 로그 적재

자동 Purge 버튼은 사용자가 쉽게 인지할 수 있도록 **상단 미리보기 영역 옆 등 눈에 잘 띄는 위치**에 배치한다.

기존 기획에서는 Metadata, ETag, MD5 변경 기반 자동 Purge를 고려했으나, 1차 개발에서는 사용자가 자동 Purge 여부를 명시적으로 선택하고 실행할 수 있는 구조를 우선한다.

---

### 2.4 Prefetch

Prefetch 기능은 지원 가능하도록 고려하되, 현재 사용 빈도가 낮으므로 1차 핵심 범위에서는 우선순위를 낮게 둔다.

- Prefetch 기능 지원 가능 구조 확보
- 1차 개발에서는 업로드/Purge 중심
- 필요 시 후속 단계에서 기능 확장

---

### 2.5 파일 비교

파일 비교 기능은 지원하되, 대용량 파일 수 처리 시 성능 이슈를 고려해야 한다.

확인 필요 사항은 다음과 같다.

- 파일 5,000개 이상 업로드 시 미권고 여부 확인 필요
- 파일 10,000개 이상 비교 시 시간이 오래 걸릴 수 있음
- 파일 10,000개 이상 업로드/비교는 미권고 방향으로 검토
- 기본 처리 단위는 1,000개 기준
- 설정을 통해 처리 단위 증가는 가능하도록 검토
- AWS 가이드상 10,000개 기준은 확인되나, 그 이상 처리 정책은 확인 필요
- 5,000개 기준 미권고 여부는 고객/운영 정책 확인 필요

---

## 3. 자동 Purge 정책

자동 Purge는 별도 체크 버튼 또는 옵션으로 제공한다.

### 기본 정책

- 기본값은 수동 Purge
- 사용자가 자동 Purge 옵션을 체크하면 업로드 이후 Purge 가능
- 자동 Purge 실행 전 승인 방식 필요
- 대용량 Purge 시 승인 또는 경고 필요
- 전체 Purge / 개별 Purge 모두 지원
- Purge Overwrite / Skip 정책 지원

### 승인 방식

자동 Purge 또는 대용량 Purge 실행 시 승인 절차가 필요하다.

검토 방향은 다음과 같다.

- 일반 업로드 후 자동 Purge 실행 전 확인 팝업
- 대용량 Purge 기준 초과 시 추가 경고
- 5,000개 이상 또는 10,000개 이상 기준은 확인 필요
- 운영자/관리자 권한에 따라 승인 정책 분리 가능성 검토

## 4. 프로필 관리

### 4.1 기본 개념

Upload Tool은 사용자가 직접 CDN/S3 Credential을 입력하는 구조가 아니라, Nexon 또는 관리자 측에서 생성한 **암호화 프로필**을 Import하여 사용하는 방식으로 구성한다.

프로필에는 다음 정보가 포함될 수 있다.

- S3 Profile 정보
- S3 접근 Credential
- CDN Provider 정보
- Purge 대상 Provider 정보
- 사용 가능 기능 권한
- 인증키
- 프로젝트 정보
- 사용자 또는 그룹 정보
- AI LB 인증 시스템에서 발급받은 키

프로필 파일은 Upload Tool에서 다시 한 번 암호화 처리하여 실행한다.

---

### 4.2 프로필 생성/배포 구조

프로필 생성은 일반 사용자가 아닌 관리자 또는 운영자 권한에서 수행하는 구조를 우선 검토한다.

기본 흐름은 다음과 같다.

1. Nexon 또는 관리자 시스템에서 프로필 생성
2. 프로필에 인증키, CDN Provider, S3 정보, 권한 정보 포함
3. 프로필 파일 전달
4. 사용자는 Upload Tool에서 프로필 Import
5. Upload Tool 내부에서 프로필 재암호화
6. 프로필 기반으로 업로드/Purge/다운로드 권한 제어

---

### 4.3 일반 사용자 권한

일반 사용자는 프로필을 직접 생성하거나 수정하지 않고, 다음 기능만 사용할 수 있도록 한다.

- 프로필 Import
- 프로필 Remove
- 프로필 선택
- 프로필 기반 업로드
- 프로필 기반 다운로드
- 프로필 기반 Purge

프로필 생성/수정/배포는 관리자 또는 운영자 권한에서 수행하는 방향으로 검토한다.

---

### 4.4 여러 프로필 관리

Upload Tool에서는 여러 프로필을 관리할 수 있어야 한다.

필요 기능은 다음과 같다.

- 여러 프로필 Import
- 프로필 검색
- 프로필 선택
- 프로필 수정 가능 여부 검토
- 프로필 Remove
- 프로필별 사용 가능 CDN Provider 표시
- 프로필별 권한 표시
- 프로젝트 단위/사용자 단위 프로필 분리 가능 구조


---

## 5. 로그 적재

Upload Tool에서 발생하는 로그는 고객이 제공하는 별도 Bucket에 누적 적재한다.

로그는 호출 또는 분석이 가능하도록 구조화된 형태로 저장한다.

### 로그 대상

로그 적재 대상 범위는 다음과 같다.

- 업로드 로그
- 다운로드 로그
- Purge 로그
- 오류 로그
- Metadata 업로드 실패 로그
- 자동 Purge 실행 로그
- Overwrite / Skip 처리 로그

### 확인 필요 사항

아래 항목은 고객 확인이 필요하다.

- 로그 적재용 Bucket
- 로그 적재 Prefix
- 로그 포맷을 JSON으로 진행해도 되는지 여부
- 로그 적재 대상 범위
- 로그 보관 기간
- 고객 측 분석 시스템 호출 방식

---

## 6. 실행 파일 및 설치 방식

Upload Tool은 설치형 또는 실행파일 방식으로 제공한다.

### 제공 방식

검토 대상은 다음과 같다.

- 설치형
- 실행파일
- Portable 방식

테스트 버전은 설치형으로 제공하고, 향후 확장성 있는 방향으로 개발한다.

---

### 지원 대상 환경

지원 대상은 다음과 같다.

### Windows Server

- Windows Server 2025
- Windows Server 2022
- Windows Server 2019

Windows Server 지원 가능 여부는 확인이 필요하다.

### Windows PC

- 최근 3개년 버전 기준 지원
- 최신 Windows 포함

### macOS

- 최근 3개년 버전 기준 지원
- 최신 macOS 포함

### Linux / Unix Server

- 최근 3개년 버전 기준 지원
- 최신 Linux 배포판 포함
- Unix 서버 지원 가능 여부 검토

## 7. 패치 및 보안 업데이트

기존 Hyper Browser 또는 기존 업로드툴에서 보안 권고 패치가 필요했던 것처럼, 신규 Upload Tool도 보안 패치 및 자동 업데이트 기능을 고려해야 한다.

### 요구사항

- 보안 권고 사항 패치 지원
- 사용자가 사용 중일 때 패치 알림
- “패치하시겠습니까?” 형태의 확인 절차
- 실행파일 재시작 없이 자동 패치 가능 여부 검토
- 자동 패치 기능 추가
- 패치 실패 시 롤백 또는 재시도 정책 검토

### 확인 필요 사항

- 무중단 패치 가능 여부
- 재시작 없는 패치가 기술적으로 가능한 범위
- 보안 패치 강제 적용 여부
- 사용자 승인 후 패치 방식
- 관리자 정책 기반 패치 방식

---

## 9. 현재 운영 흐름 참고

현재 파일 전달 및 Purge 흐름은 다음과 같이 정리된다.

1. 현재 파일을 아카이브 형태로 전달
2. Purge Tool에서 처리
3. 업로드 또는 Purge 수행
4. 로그 생성
5. 필요 시 고객 Bucket에 로그 적재

신규 Upload Tool에서는 해당 흐름을 프로필 기반, UI 기반, 로그 적재 기반으로 통합한다.

## 11. 확인 필요 항목

### 11.1 대용량 처리 기준

- 5,000개 이상 업로드 시 미권고 기준이 맞는지
- 10,000개 이상 파일 비교/업로드/Purge 미권고 기준이 맞는지
- 기본 Batch Size를 1,000개로 둘지
- 사용자가 Batch Size를 늘릴 수 있게 할지
- AWS/CDN Provider별 Purge 제한 기준
- 대용량 Purge 시 승인 방식

---

### 11.2 로그 적재

- 로그 적재용 Bucket
- 로그 적재 Prefix
- 로그 포맷 JSON 가능 여부
- 로그 적재 범위
- 로그 보관 기간
- 고객 분석 시스템 연동 방식

---

### 11.3 프로필 기준

- 프로필을 프로젝트 단위로 분리할지
- 프로필을 사용자 단위로 분리할지
- 하나의 프로필에 복수 CDN Provider 정보를 포함할 수 있는지
- 일반 사용자는 Import/Remove만 가능하면 되는지
- 프로필 생성/수정/배포 권한은 관리자/운영자만 가지면 되는지
- 인증키 발급 주체

---

### 11.4 설치 및 실행 환경

- 설치형/실행파일/Portable 중 최종 제공 방식

- macOS 최근 3개년 버전 지원 범위
- Linux 서버 최근 3개년 버전 지원 범위
