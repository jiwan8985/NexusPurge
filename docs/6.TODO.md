# UploadTool TODO

## TODO 개요

UploadTool의 TODO는 **고객사 확인 필요 항목**, **1차 개발 TODO**, **후속 고도화 TODO**로 구분합니다.

1차 개발은 기존 CDN 업로드/Purge 툴의 기능 재현을 목표로 하며, 고객사 확인이 필요한 항목은 확정 전까지 TODO로 관리합니다.

## 정리

> UploadTool TODO는 우선 **기존 툴 기능 재현을 위한 문서/구조 정리**와 **CDN Provider 확장**에 집중합니다.
> 
> 
> LG U+ CDN, 효성 CDN, KT CDN은 API 문서 수령 전까지 Stub Adapter로 구성하고, 실제 Purge API는 고객사 확인 후 구현합니다.
> 
> 인증은 자체 구현하지 않고 외부 인증 모듈 연동 구조만 준비하며, 작업 로그/결과 처리 구조는 1차 개발 범위에 포함합니다.
>
