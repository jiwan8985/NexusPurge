# NexusPurge (CDN Upload Tool) 사용자 가이드

S3 업로드와 CDN 캐시 무효화(Purge)를 한 화면에서 처리하는 Windows 데스크톱 도구입니다.

FTP 클라이언트처럼 좌(내 PC) / 우(S3 버킷) 듀얼 패널로 파일을 올리고, 업로드·삭제 시 CDN 캐시를 자동으로 갱신합니다.

- 지원 CDN: **AWS CloudFront · Akamai · LG U+ CDN · KT CDN · 효성 ITX**
- 한 프로필에 여러 CDN을 함께 구성할 수 있으며, 선택된 CDN 전체에 동시에 Purge합니다.

---

## 1. 설치

### 시스템 요구사항
- Windows 10 / 11 (64-bit)
- 인터넷 연결 (S3 및 CDN API 접근)

### 설치 방법
1. 전달받은 설치 파일(`.msi`)을 실행합니다.
2. 설치 마법사의 안내에 따라 설치를 완료합니다.
3. 시작 메뉴 또는 바탕화면에서 앱을 실행합니다.

> 자격증명(Secret Key 등)은 파일이 아닌 **Windows 자격 증명 관리자(Credential Manager)** 에 암호화 저장됩니다.

---

## 2. 시작하기 — 프로필 등록과 연결

프로필은 S3 버킷 + CDN 접속 정보 묶음입니다. 
**직접 입력은 지원하지 않으며, 전달받은 프로필 파일을 가져오는 방식만 사용합니다.**

### 2-1. 프로필 가져오기
1. 툴바의 **[프로필]** 버튼 → 접속 프로필 관리 창
2. **[↓ 가져오기]** 클릭
3. 전달받은 파일 선택:
   - **`.json`** — 일반 프로필 파일. 
   - **`.nexprofile`** — 암호화 프로필 파일. 가져오기 시 **내보낼 때 사용한 패스프레이즈**를 입력해야 합니다.
4. 목록에 프로필 이름이 나타나면 완료

> **주의**: 암호화 파일 여부는 **확장자(`.nexprofile`)로 판별**합니다. 파일 이름을 변경해 확장자가 바뀌면 가져오기가 실패하니, 내보내기 시 생성된 파일명을 그대로 사용하세요.

### 2-2. 연결
- 프로필 행의 **[연결]** 을 누르면 S3에 접속하고 우측 패널에 버킷 내용이 표시됩니다.
- 연결 시 프로필에 구성된 **모든 CDN이 Purge 대상으로 자동 활성화**됩니다 (툴바에서 개별 해제 가능).

### 2-3. 테스트
- 프로필 행의 **[테스트]** 는 저장된 자격증명으로 S3 연결과 구성된 각 CDN 연결을 검사해 ✓/✗ 결과만 보여줍니다.
- 연결이 안 될 때 원인(S3 문제인지, 특정 CDN 문제인지)을 빠르게 구분할 수 있습니다.

### 2-4. 프로필 정보 열람 불가 (보안 정책)
- 저장된 프로필은 **이름만 표시**되며, 버킷·키·CDN 설정 등 어떤 값도 다시 볼 수 없습니다.
- 가능한 작업: **연결 · 테스트 · 내보내기(암호화 파일 재발급) · 삭제**
- 설정 변경이 필요하면 프로필을 **삭제한 뒤 새 프로필 파일을 다시 가져오세요.** (새 파일은 프로필 발급 담당자에게 요청)

## 3. 화면 구성

```
┌──────────────────────────────────────────────────────┐
│ 툴바: 프로필 | 새 폴더 | 삭제 | 이름변경 | 자동Purge |     │
│       CDN 선택 칩 | 수동 Purge | 설정                   │
├──────────────────┬───────┬───────────────────────────┤
│  로컬 파일 패널     │ 전송   │  S3 버킷 패널               │
│  (내 PC)          │ 버튼   │  (원격)                    │
├──────────────────┴───────┴───────────────────────────┤
│  작업 로그 패널 (시스템/전송/CDN 탭, 로그 폴더 열기)        │
├──────────────────────────────────────────────────────┤
│  상태바                                                │
└──────────────────────────────────────────────────────┘
```

- **로컬 패널**: 내 PC 탐색. 경로 직접 입력, ↑(상위 폴더), 더블클릭으로 폴더 이동
- **S3 패널**: 버킷 탐색. 이름/크기/수정일/ETag 표시, ↻ 새로고침
- 행 클릭 = 선택, `Ctrl+클릭` = 다중 선택, 더블클릭 = 폴더 열기

## 4. 파일 업로드

업로드 방법은 세 가지이며 결과는 동일합니다.

| 방법 | 사용법 |
|---|---|
| 전송 버튼 | 로컬 패널에서 파일/폴더 선택 → 가운데 **→** 버튼 |
| 패널 간 드래그 | 로컬 패널의 파일/폴더 행을 잡아 **S3 패널에 드랍** |
| 탐색기 드래그 | **Windows 탐색기**에서 파일/폴더를 앱의 S3 패널로 드랍 |

- **폴더**를 올리면 하위 파일 전체가 폴더 구조 그대로 업로드됩니다.
- 업로드 위치는 S3 패널에 **현재 열려 있는 경로**입니다.
- 드래그 중 대상 패널이 강조되고, `Esc` 로 드래그를 취소할 수 있습니다.

### 4-1. Smart Sync (중복 업로드 방지)
업로드 전에 로컬 파일의 MD5와 S3의 ETag를 비교해 자동 분류합니다.

| 뱃지 | 의미 | 동작 |
|---|---|---|
| **신규** | S3에 없는 파일 | 업로드 |
| **스킵** | S3와 내용 동일 | 업로드 생략 (트래픽 절약) |
| **수정/Purge** | S3와 내용 다름 | 덮어쓰기 업로드 + CDN Purge 대상 |

### 4-2. 진행률 창
- 파일별 진행률/속도/남은 시간과 전체 N/M 카운트를 표시합니다. 카운트는 **이번 업로드 배치 기준**입니다.
- 창을 닫아도(창 숨기기) 전송은 백그라운드에서 계속됩니다.
- 개별 파일 **[취소]** 가능. 완료 후 업로드된 파일들의 **CDN URL 목록 복사** 제공.

## 5. 파일 다운로드

- S3 패널에서 파일/폴더 선택 → 가운데 **←** 버튼, 또는 S3 행을 **로컬 패널로 드래그**, 또는 우클릭 → **다운로드**
- 저장 폴더 선택 창이 뜨며, 폴더 다운로드 시 하위 구조가 그대로 유지됩니다.

---

## 6. S3 파일 관리

- **새 폴더**: 툴바 [새 폴더] (S3 패널 포커스 상태)
- **이름 변경**: 우클릭 → 이름 변경, 또는 `F2`
- **삭제**: 우클릭 → 삭제, 또는 `Delete` — 삭제 성공한 객체는 활성 CDN에서 자동 Purge됩니다
- 폴더/파일 이름에는 S3 안전 문자만 허용됩니다: 한글/영문/숫자와 `! - _ . * ' ( )` (공백, `/ \ ? # % & +` 등 불가)

## 7. CDN Purge (캐시 무효화)

### 7-1. Purge 대상 CDN 선택
- 툴바의 **CDN 칩**에서 Purge 대상 CDN을 켜고 끕니다. **연결 직후에는 프로필의 모든 CDN이 켜진 상태**입니다.
- 여러 CDN이 켜져 있으면 모든 Purge가 해당 CDN들에 **동시에** 실행됩니다.

### 7-2. 자동 Purge
- 툴바 **자동 Purge ON/OFF** 토글.
- ON이면 업로드 완료 후(덮어쓰기·신규 포함) 해당 경로들이 자동으로 Purge됩니다. 삭제 시에도 삭제된 경로가 Purge됩니다.

### 7-3. 수동 Purge
| 대상 | 방법 |
|---|---|
| 파일 1개 | S3 패널에서 우클릭 → **Purge** |
| **폴더 전체** | 폴더 우클릭 → **Purge (폴더 전체)** — 폴더 안 **모든 하위 파일**(1,000개 이상 포함)이 무효화됩니다 |
| 선택 항목 | 여러 항목 `Ctrl+클릭` 후 툴바 수동 Purge |
| 전체 | 툴바에서 현재 경로 전체(`…/*`) Purge |
| 경로 직접 입력 | 툴바 → CDN 경로 직접 입력 Purge |

### 7-4. 폴더 Purge의 CDN별 처리 방식 (자동 — 참고용)
| CDN | 처리 |
|---|---|
| CloudFront | `폴더/*` 와일드카드 무효화 1건 |
| Akamai | CP Code 무효화 (미설정 시 개별 URL) |
| LG U+ / KT | S3 목록을 조회해 **개별 파일 경로로 자동 확장** 후 Purge. 버킷 루트 전체 + cloudcdn 타입이면 서비스 전체 즉시 플러시 |
| 효성 ITX | S3 목록을 조회해 **개별 파일 경로로 자동 확장** 후 Purge |

- 대량 경로는 설정의 **Purge 배치 크기** 단위로 분할 전송되며, 결과 창에서 CDN별 성공/실패와 소요 시간을 확인할 수 있습니다.
- 한글/공백 파일명은 자동으로 URL 인코딩되어 전송됩니다.

## 8. 객체 속성 · CDN 실시간 확인

S3 패널에서 파일 우클릭 → **속성**:

- **기본 정보**: S3 Key(복사 버튼), 버킷, 리전, 크기, 수정일, ETag, Content-Type
- **S3 응답 헤더 (실시간)**: [조회] 버튼 — HeadObject 전체 응답 (Cache-Control, 암호화, 사용자 메타데이터 등)
- **CDN 정보** (CDN별 카드):
  - Edge Domain, Service/Distribution ID, **CDN URL** (복사 버튼), Purge 요청 엔드포인트
  - **[실시간 확인]** — 실제 공개 URL로 요청해 HTTP 상태코드와 응답 헤더를 표시 (캐시 반영 여부 확인용)

### 실시간 확인 오류 메시지 읽는 법
| 메시지 | 의미 | 조치 |
|---|---|---|
| **DNS 조회 실패 … (NXDOMAIN)** | CDN 도메인이 아직 DNS에 등록되지 않음. 브라우저에서도 `ERR_NAME_NOT_RESOLVED`로 열리지 않음 | **앱 문제 아님** — CDN 도메인 관리 부서에 서비스 도메인 DNS(CNAME) 등록 요청 |
| 응답 시간 초과 | 서버 무응답 (방화벽/사설망 가능성) | 네트워크/방화벽 확인 |
| 연결 실패 | 포트 차단 또는 서버 다운 | 방화벽 아웃바운드 확인 |
| TLS 인증서 오류 | 인증서 체인 문제 | CDN 사업자 문의 |

## 9. 작업 로그

- 하단 **로그 패널**: 시스템/전송/CDN 카테고리별 실시간 로그. Purge 로그에는 시작·종료 시각과 소요 시간이 표시됩니다.
- **[로그 폴더]** 버튼으로 파일 로그 폴더가 열립니다 (`%LOCALAPPDATA%\cdn-upload-tool\logs\`):

| 파일 | 내용 |
|---|---|
| `system-YYYY-MM-DD.log` | 폴더 생성/이름 변경/삭제 등 파일 관리 작업 |
| `transfer-YYYY-MM-DD.log` | 업로드/다운로드 결과 |
| `cdn-YYYY-MM-DD.log` | CDN Purge 요약 (CDN별 상태, 요청 엔드포인트, 소요시간, 대상 경로, 오류 전문) |
| `audit.YYYY-MM-DD.log` | **CDN API 상세 감사 로그** — 인증·Purge·상태조회 등 모든 CDN API 호출의 메서드/URL/**HTTP 상태(예: 200 OK)**/소요시간/응답 본문, 폴더 Purge 확장 내역, 재시도 이력. "왜 Purge가 안 됐는지" 추적할 때 가장 먼저 확인 |
| `operation_logs.json` | 전체 작업 이력 원본 (경로 목록 무제한 보관 — 감사용) |

장애 문의 시 해당 날짜의 로그 파일을 함께 전달해 주시면 원인 파악이 빠릅니다.

## 10. 설정 (툴바 → 설정)

| 항목 | 설명 |
|---|---|
| 마지막 프로필 선택 복원 | 앱 시작 시 마지막 연결 프로필을 선택 상태로 표시 |
| 시작 시 로그 패널 표시 | 실행 시 하단 로그 패널 기본 표시 여부 |
| 동시 전송 수 | 업로드/다운로드 병렬 개수 (1~16, 기본 8) |
| 현재 로그 패널 표시 | 현재 화면의 로그 패널 표시/숨김 |
| 프로필 관리 / 완료 전송 정리 / 로그 지우기 | 바로가기 버튼 |

> 대량 선택 경고, 대용량 파일 확인, Purge 경고, Purge 배치 크기 등은 내부적으로 안전한 기본값(파일 5,000/10,000개, 100MB, Purge 1,000개 기준)으로 자동 동작하며 별도 설정이 필요 없습니다.
