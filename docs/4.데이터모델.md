# UploadTool 데이터 모델

## 데이터 모델 개요

UploadTool의 데이터 모델은 크게 **CDN Provider**, **S3 Profile**, **외부 인증 연동**, **작업 로그/결과**, **Backend Credential** 구조로 구분됩니다.

본 데이터 모델은 기존 CDN 업로드/Purge 툴의 기능을 재현하면서, 향후 CDN별 API 확장과 외부 인증 모듈 연동을 고려한 구조입니다.

---

## CdnProvider

CDN Provider를 구분하기 위한 타입입니다.

| 값 | 설명 |
| --- | --- |
| `cloudfront` | AWS CloudFront |
| `akamai` | Akamai CDN |
| `lguplus` | LG U+ CDN |
| `hyosung` | 효성 CDN |
| `kt` | KT CDN |

---

## S3Profile

S3Profile은 AWS S3 접속 정보와 CDN Purge 설정을 함께 관리하는 Profile 모델입니다.

| 필드 | 설명 |
| --- | --- |
| `id` | Profile 고유 ID |
| `name` | Profile 이름 |
| `region` | AWS Region |
| `bucket` | AWS S3 Bucket 이름 |
| `basePrefix` | 기본 S3 Prefix |
| `accessKeyId` | AWS Access Key ID |
| `secretAccessKey` | AWS Secret Access Key |
| `cdnProvider` | CDN Provider |
| `cdnDistributionId` | CloudFront Distribution ID |
| `cdnDomain` | CDN 도메인 |
| `akamaiClientToken` | Akamai Client Token |
| `akamaiClientSecret` | Akamai Client Secret |
| `akamaiAccessToken` | Akamai Access Token |
| `akamaiHost` | Akamai API Host |
| `lguplusApiKey` | LG U+ CDN API Key |
| `lguplusApiSecret` | LG U+ CDN API Secret |
| `lguplusEndpoint` | LG U+ CDN API Endpoint |
| `hyosungApiKey` | 효성 CDN API Key |
| `hyosungApiSecret` | 효성 CDN API Secret |
| `hyosungEndpoint` | 효성 CDN API Endpoint |
| `ktApiKey` | KT CDN API Key |
| `ktApiSecret` | KT CDN API Secret |
| `ktEndpoint` | KT CDN API Endpoint |
| `purgeOnNewUpload` | 신규 업로드 파일 Purge 여부 |
| `defaultCacheControl` | 기본 Cache-Control 값 |
| `contentTypeOverride` | Content-Type 강제 지정 값 |

## 데이터 모델 정리

> UploadTool의 데이터 모델은 **Profile 중심 구조**로 설계합니다.
> 
> 
> S3 접속 정보, CDN Provider 설정, Purge 정책, 인증 사용자 정보, 작업 로그를 분리하여 관리합니다.
> 
> Frontend에서는 TypeScript 타입으로 관리하고, Backend에서는 Rust Credential 및 작업 로그 구조로 분리하여 처리합니다.
>
