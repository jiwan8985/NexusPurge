시나리오 25 — 단일 파일 업로드 + Purge (용량별)
================================================================

목적: 용량이 다른 단일 파일을 각각 업로드하고 Purge하여
      크기별 처리 흐름과 경고 팝업이 정상 동작하는지 확인

[파일 목록]
tiny-under-1kb.txt        : ~50 bytes   (최소 크기)
small-10kb.txt            : ~10 KB      (일반 텍스트/설정 파일 수준)
medium-100kb.txt          : ~100 KB     (중간 크기)
large-5mb.txt             : ~5 MB       (대용량 경고 임계값 미만 - 기본 100MB)
xlarge-110mb-PLACEHOLDER  : 110 MB 생성 스크립트 (git 미포함, 로컬 생성 필요)

[테스트 A — 소/중 용량 단일 파일 업로드]
1. tiny, small, medium 파일 각각 개별 업로드
2. 대용량 확인 팝업 없이 바로 업로드 시작 확인
3. 각 파일 업로드 완료 후 [선택 Purge]
4. LogPanel에서 각 파일 상태 확인

[테스트 B — 대용량 경고 트리거]
설정 → 대용량 파일 확인 기준을 4MB로 낮춤
1. large-5mb.txt 업로드 시도
2. 대용량 업로드 확인 팝업 등장 확인:
   "5개 파일, 총 X MB를 업로드합니다."
3. [옵션 설정 후 업로드] 또는 취소

[테스트 C — 110MB 파일 업로드 (기본 임계값 초과)]
1. 로컬에서 110MB 파일 생성 (PLACEHOLDER 파일 내 스크립트 참고)
2. 업로드 시도 → 기본값(100MB) 초과로 대용량 팝업 표시
3. Multipart 업로드 처리 확인 (8MB 이상은 자동 Multipart)
4. 업로드 완료 후 Purge 실행

[기대 결과]
tiny~medium: 팝업 없이 즉시 업로드 ✓
large-5mb  : 임계값 낮출 경우 대용량 팝업 ✓
xlarge-110mb: 기본값(100MB) 초과 → 팝업 ✓
             Multipart 업로드 처리 ✓
