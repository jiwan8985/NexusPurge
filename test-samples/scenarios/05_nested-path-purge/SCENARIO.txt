================================================================
시나리오 05 — 중첩 경로(Nested Path) Purge
================================================================

목적: 서로 다른 깊이의 경로에 있는 파일들을 Purge할 때
      경로가 정확하게 전달되는지 확인

[S3 업로드 구조]
contents/scenarios/05_nested-path-purge/
├── root-file.txt             (루트)
└── depth1/
    ├── file.txt              (1단계)
    └── depth2/
        ├── file.txt          (2단계)
        └── depth3/
            └── file.txt      (3단계)

[테스트 A — 특정 깊이만 Purge]
1. 모든 파일 업로드
2. depth1/file.txt 만 선택 → [선택 Purge]
3. root-file.txt, depth2/file.txt, depth3/file.txt 는 캐시 유지 확인

[테스트 B — 특정 하위 경로 전체 Purge]
1. 원격 패널에서 depth1/ 경로로 이동
2. [전체 Purge] 실행 → depth1/* 패턴 적용
3. depth1 하위 모든 파일 무효화, root-file.txt 는 유지

[테스트 C — 루트에서 전체 Purge]
1. 원격 패널을 05_nested-path-purge/ 경로로 유지
2. [전체 Purge] 실행 → 05_nested-path-purge/* 패턴
3. 하위 중첩 경로까지 모두 무효화되는지 확인

[CDN 별 동작 차이]

CloudFront: /* 패턴은 1단계만 적용 (depth2 이하는 별도 경로 필요)
