시나리오 21 — 업로드 없이 Purge만 반복 실행
================================================================

목적: 파일을 업로드하지 않고 Purge만 반복해서 실행하는 비정상 사용 패턴
      운영자가 "캐시가 안 날아간 것 같아서" 계속 Purge 버튼을 누르는 상황

[파일 없음 — Purge만 실행]

[테스트 A — 아무것도 선택 안 하고 Purge]
1. 원격 패널에서 아무 파일도 선택하지 않음
2. [선택 Purge] 버튼 상태 확인

[기대 결과 A]
- [선택 Purge] 버튼 비활성화(disabled)
- 선택 없이 버튼 활성화되면 안 됨

[테스트 B — 전체 Purge 5회 연속 실행]
1. [전체 Purge] 버튼을 빠르게 5회 연속 클릭
2. PurgeDialog가 5개 중첩으로 뜨는지 확인 또는 한 번만 처리되는지 확인

[기대 결과 B]
- 중복 팝업 방지 (최초 1개만 표시)
- 또는 5번 모두 독립 처리되더라도 오류 없이 정상 동작

[테스트 C — Purge 완료 후 즉시 재클릭 (버튼 연타)]
1. [선택 Purge] → PurgeDialog → 확인 클릭
2. Purge 실행 중(isPurging=true 상태) 다시 [선택 Purge] 클릭

[기대 결과 C]
- Purge 실행 중에는 버튼 비활성화 또는 중복 실행 방지

[테스트 D — 빈 원격 패널에서 전체 Purge]
1. S3에 아무 파일도 없는 경로로 이동
2. [전체 Purge] 실행

[기대 결과 D]
- 빈 경로에서도 Purge API 호출 정상 처리
- 오류 없이 "Purge 완료" 기록
