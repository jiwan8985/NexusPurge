시나리오 22 — 동일 파일 수십 번 반복 업로드 + Purge
================================================================

목적: "Purge 됐나?" 확인 목적으로 같은 파일을 계속 올리고 Purge하는 패턴
      운영팀에서 매우 흔한 비정상 사용 패턴

[배경]
게임 운영자의 실제 행동 패턴:
1. 파일 업로드
2. CDN에서 확인 → 안 바뀐 것 같음 (캐시 때문에)
3. 다시 업로드
4. 또 Purge
5. 반복...

[순서 — 10회 연속 동일 파일 업로드+Purge]
1. repeat-target.txt 업로드
2. [선택 Purge] 실행
3. 파일 수정 없이 즉시 1, 2 반복 (10회)

[기대 결과]
- 매 업로드 시 Smart Sync: "크기 동일 → Skip" 처리
- 매 Purge는 정상 실행
- LogPanel에 10회 이력 기록
- 불필요한 S3 업로드 비용 발생 없음 (Skip이므로)

[실제 문제 상황]
Smart Sync가 Skip으로 처리했지만
운영자는 "업로드가 안 된 것" 으로 오해하고 계속 재시도
→ UI에서 Skip 이유를 명확하게 안내하는 것이 중요

[검증]
LogPanel 전송 큐 탭에서 10회 중:
- 최초 1회: complete (신규 업로드)
- 이후 9회: skipped (동일 크기 → Skip)
모두 Purge는 정상 실행됨
