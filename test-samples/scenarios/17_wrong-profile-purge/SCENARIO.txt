시나리오 17 — 잘못된 프로필/자격증명으로 Purge 시도
================================================================

목적: 권한이 없거나 잘못된 설정의 프로필로 Purge 시도 시
      앱이 오류를 명확하게 표시하는지 확인

[테스트 A — CDN 미설정 프로필로 Purge 시도]
1. CDN Provider가 설정되지 않은 S3 전용 프로필 연결
2. 파일 업로드 후 원격 패널에서 파일 선택
3. Purge 버튼 확인

[기대 결과 A]
- Purge 버튼이 비활성화(disabled)되어야 함
- "CDN Provider가 설정되지 않은 프로필" 툴팁 표시

[테스트 B — 만료/잘못된 CDN 자격증명으로 Purge]
1. CDN Provider는 설정되어 있으나 ACCESS_TOKEN 값을 일부러 잘못 입력한 프로필 생성
2. 파일 업로드 후 [선택 Purge] 실행

[기대 결과 B]
- Purge API 호출 후 401/403 오류 수신
- LogPanel: "Purge 실패" 오류 메시지 + 오류 상세
- 전송 큐: CDN Purge 상태 "오류" 표시
- 앱 크래시 없이 오류 처리

[테스트 C — 잘못된 Distribution ID로 CloudFront Purge]
1. Distribution ID를 존재하지 않는 ID로 변경 (예: INVALID123)
2. [선택 Purge] 실행

[기대 결과 C]
- CloudFront API: NoSuchDistribution 오류
- LogPanel: 구체적인 오류 내용 표시
- 사용자가 원인을 파악할 수 있는 메시지 제공

[테스트 D — Akamai CP_CODE 오류]
1. CP_CODE를 잘못된 값으로 변경
2. [선택 Purge] 실행

[기대 결과 D]
- Akamai EdgeGrid API 오류 응답
- LogPanel에 오류 기록
