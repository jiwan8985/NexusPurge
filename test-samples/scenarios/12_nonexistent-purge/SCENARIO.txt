시나리오 12 — S3에 없는 파일 Purge (존재하지 않는 경로)
================================================================

목적: S3에 업로드하지 않은(또는 이미 삭제된) 파일의 CDN 경로를 Purge할 때
      오류 없이 처리되는지 확인
      게임사에서 "혹시 캐시 남아있을 수 있으니 Purge" 하는 상황

[파일 없음 — 업로드 없이 바로 Purge 시도]

[테스트 A — 존재하지 않는 단일 경로 Purge]
1. 아무 파일도 업로드하지 않음
2. 원격 패널에서 임의 경로 직접 입력 후 [선택 Purge]
   예: contents/scenarios/12_nonexistent-purge/ghost-file.txt
3. CDN에 캐시 없는 경로에 Purge 요청

[테스트 B — 삭제된 파일 Purge]
1. ghost-file.txt 를 먼저 업로드 (CDN 캐시 생성)
2. S3에서 ghost-file.txt 삭제
3. CDN에는 캐시 남아있는 상태
4. [선택 Purge] 실행 → CDN 캐시 강제 제거

[테스트 C — 오타 경로 Purge]
1. 실제 파일: contents/patch/v2.0/update.txt
   오타 경로: contents/patch/v2.0/updaet.txt (오타)
2. 오타 경로로 Purge 실행
3. 실제 파일 캐시는 유지되어야 함

[기대 결과]
A: CDN API는 성공 응답 (존재 여부 관계없이 경로 무효화 요청 처리)
B: CDN 캐시 제거 성공 → 이후 CDN 접근 시 오리진(S3)으로 요청 → 404
C: 오타 경로만 무효화, 실제 파일 캐시 유지

[주의]
Akamai / CloudFront 모두 경로 존재 여부 관계없이 Purge API 성공 응답
실제 캐시 없는 경로 Purge는 무해하지만 API 호출 비용은 발생할 수 있음
