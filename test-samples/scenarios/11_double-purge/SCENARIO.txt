시나리오 11 — 이미 Purge된 파일 중복 Purge (Double Purge)
================================================================

목적: Purge 완료 직후 동일 경로를 즉시 재Purge 했을 때 오류 없이 처리되는지 확인
      게임사 운영팀이 "됐나?" 확인하려다 Purge 버튼을 두 번 누르는 상황

[순서]
1. target.txt 업로드
2. [선택 Purge] 실행 → Purge 완료 확인
3. 완료 즉시 다시 [선택 Purge] 실행 (캐시 없는 상태에서 Purge)
4. 세 번째도 반복 실행

[기대 결과]
- 2, 3차 Purge 모두 오류 없이 API 응답 200 처리
- CloudFront: caller_reference 중복 → 멱등 처리 (동일 Invalidation 반환)
- Akamai: 캐시 없는 상태의 Purge 요청도 정상 처리
- LogPanel에 오류 없이 "완료" 기록

[실패 케이스]
- CloudFront InvalidationBatchAlreadyExists 오류 발생 시
  → caller_reference 생성 로직 점검 필요
