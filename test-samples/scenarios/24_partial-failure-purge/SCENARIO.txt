시나리오 24 — Purge 부분 성공 / 부분 실패
================================================================

목적: 여러 파일 Purge 중 일부는 성공하고 일부는 실패했을 때
      앱이 결과를 정확하게 표시하는지 확인

[재현 방법]

[방법 A — 유효 + 무효 경로 혼합 Purge]
1. file-ok-1.txt, file-ok-2.txt 업로드 (실존 경로)
2. 원격 패널에서 두 파일 선택
3. Purge 실행 직전 S3에서 file-ok-2.txt 삭제
4. CDN에 캐시는 있지만 S3에는 없는 상태에서 Purge

[기대 결과 A]
- 두 경로 모두 Purge API 호출 (존재 여부 무관)
- CDN: 두 경로 모두 캐시 무효화
- S3에 없어도 Purge는 성공 (CDN 기준)

[방법 B — 네트워크 불안정 상황 시뮬레이션]
1. VPN 또는 네트워크를 끊었다 연결했다 하며 Purge 실행
2. Purge 중 타임아웃 발생 시 앱 동작 확인

[기대 결과 B]
- 타임아웃 오류를 LogPanel에 명확히 표시
- 실패 항목 탭에 기록
- 앱 크래시 없이 오류 처리

[방법 C — CloudFront Throttling 유도]
1. 짧은 시간에 매우 많은 Invalidation 요청 전송
   (CloudFront 기본 한도: 초당 1,500개 경로)
2. ThrottlingException 발생 시 앱 처리 확인

[기대 결과 C]
- ThrottlingException → LogPanel에 구체적 오류 메시지
- 자동 재시도 또는 사용자에게 재시도 안내
