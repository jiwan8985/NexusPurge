[테스트 A — 5,000개 경고 임계값 확인]
기본 설정: 파일 수 경고 기준 = 5,000개

1. stress-5000/ 폴더 전체 선택 (5,000개)
2. 업로드 버튼 클릭
3. 경고 팝업 확인:
   "선택한 파일이 5,000개입니다. 배치 처리 시간이 매우
    오래 걸릴 수 있습니다. 계속하시겠습니까?"
4. 계속 선택 후 업로드 진행
5. LogPanel에서 전체 진행률 확인

[테스트 B — 10,000개 제한 임계값 확인]
기본 설정: 파일 수 제한 기준 = 10,000개

1. stress-10000/ 폴더 전체 선택 (10,000개)
2. 업로드 버튼 클릭
3. 강한 경고 팝업 확인:
   "10,000개 이상의 파일은 S3/CDN 처리 한계에 도달할 수 있습니다."
4. 업로드 실행 → 동시 전송 수 16으로 설정 권장

[테스트 C — 10,000개 업로드 후 전체 Purge]
1. stress-10000/ 업로드 완료
2. [전체 Purge] 실행
3. Purge 배치 처리 확인:
   - Purge 배치 크기 기본 1,000개
   - 10,000개 → 10회 배치로 분할 처리
   - 각 배치 결과 LogPanel에 표시 확인
4. CloudFront: InvalidationBatch 최대 3,000경로 확인
   → 10,000개는 4회 API 호출로 처리

[테스트 D — 5,000개 Smart Sync 재업로드]
1. 전체 업로드 완료 후 일부(500개) 파일 내용 수정
2. 전체 5,000개 다시 선택 → 업로드
3. Smart Sync 결과 확인:
   - 변경 500개 → toOverwrite → 자동 Purge
   - 미변경 4,500개 → toSkip
4. Skip 비율이 90%일 때 속도 개선 확인

[테스트 E — 동시 전송 수 조정으로 성능 비교]
stress-5000/ 기준:
  동시  4개 (기본): 예상 소요 시간 기록
  동시 16개        : 예상 소요 시간 기록
  동시 32개 (최대) : 예상 소요 시간 기록
  → 병목이 Rust 스레드인지 네트워크인지 확인
