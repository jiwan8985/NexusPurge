시나리오 18 — 극단적 경로/파일명 (S3 키 한계 테스트)
================================================================

목적: S3 키 최대 길이(1024바이트) 근접 경로나 매우 긴 파일명에서
      업로드 및 Purge가 정상 처리되는지 확인

[S3 제한]
- 객체 키 최대 길이: 1,024바이트
- 단일 경로 컴포넌트 최대: OS에 따라 255자

[파일 구성]
very-long-filename-xxx...xxx.txt : 200자 파일명
a/b/c/d/e/f/g/h/i/j/deep-file   : 10단계 깊이 경로
path-test/aaa.../file.txt        : 50자 prefix 폴더명

[테스트 A — 긴 파일명 업로드 + Purge]
1. 200자 파일명 파일 업로드
2. 원격 패널에서 파일 표시 확인 (잘림 없이 전체 표시 여부)
3. [선택 Purge] 실행
4. CDN Purge 경로가 전체 파일명으로 전달되는지 확인

[테스트 B — 10단계 중첩 경로 Purge]
1. a/b/c/d/e/f/g/h/i/j/deep-file.txt 업로드
2. 원격 패널에서 각 단계 탐색 가능한지 확인
3. 해당 파일 선택 후 [선택 Purge]
4. CDN 경로가 전체 경로로 올바르게 전달되는지 확인

[테스트 C — 전체 경로 길이 1,000바이트 근접]
S3 업로드 prefix: contents/scenarios/18_extreme-long-path/path-test/aaa.../
파일명: file.txt
→ 합산 경로 길이 확인

[기대 결과]
- 모든 파일 정상 업로드 및 Purge 처리
- 경로 잘림 없이 전체 키 전달
- UI에서 긴 파일명 말줄임(...) 표시하되 실제 경로는 full path

[실패 시 확인]
- S3 PutObject 400 오류 → 키 길이 초과
- CDN Purge 경로 truncation → 잘린 경로로 캐시 미스 실패
