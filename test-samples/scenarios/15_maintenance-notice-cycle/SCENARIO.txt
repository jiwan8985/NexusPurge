시나리오 15 — 점검 공지 반복 교체 (게임사 일상 패턴)
================================================================

목적: 점검 시간 연장 등으로 공지 파일을 단시간에 여러 번 교체하고
      매번 즉시 Purge해야 하는 게임사 일상 운영 상황

[배경]
게임 점검 중 발생하는 실제 패턴:
1. 점검 공지 최초 게시 → CDN 배포
2. 점검 연장 → 공지 수정 → 즉시 Purge 필요
3. 재연장 → 또 수정 → 또 Purge
4. 점검 완료 공지 → 교체 → Purge
→ 짧은 시간(수 시간) 내에 동일 파일 4회 교체 + Purge

[파일 구성]
notice-v1.txt  : 최초 점검 공지 (06:00~08:00)
notice-v2.txt  : 1차 연장 (06:00~10:00)
notice-v3.txt  : 재연장 (06:00~12:00)
notice-end.txt : 점검 완료 공지

[순서]
1. notice-v1.txt 를 notice.txt 이름으로 업로드 + CDN 확인
2. (수십 분 후) notice-v2.txt 를 notice.txt 로 교체 + 즉시 Purge
3. (또 수십 분 후) notice-v3.txt 를 notice.txt 로 교체 + 즉시 Purge
4. (점검 완료) notice-end.txt 를 notice.txt 로 교체 + 즉시 Purge

[빠른 테스트 (시간 없이 연속 실행)]
1~4 단계를 기다림 없이 연속으로 실행하여
단시간 내 동일 경로 4회 연속 Purge 동작 확인

[기대 결과]
- 매 교체마다 Overwrite 분류 + Purge 실행
- 각 Purge 후 CDN에서 최신 공지 내용 반영
- LogPanel에 4회 교체 이력 모두 기록
- CDN에 이전 공지가 노출되지 않음 (Purge 지연 최소화)
