================================================================
시나리오 28 — 크기가 제각각인 파일 혼합 업로드 + Purge
================================================================

목적: 실제 게임 CDN 운영처럼 용량이 극단적으로 다른 파일들을
      한 번의 업로드로 처리할 때 모든 파일이 정상 처리되는지 확인
      특히 소형 파일과 Multipart 업로드 대상 파일이 함께 처리되는 경우

[파일 크기 분포]
version.json.txt    : 200 bytes   (설정 파일, 자주 배포)
server-config.txt   : 2 KB        (서버 설정)
patch-notes.txt     : 10 KB       (패치 노트)
item-table.txt      : 100 KB      (아이템 DB 스냅샷)
quest-data.txt      : 500 KB      (퀘스트 데이터)
npc-dialogue.txt    : 1 MB        (NPC 대화 텍스트)
world-map-meta.txt  : 5 MB        (월드맵 메타데이터)

mixed-batch/
  tiny-1~5.txt      : ~10 bytes 각 (초소형 5개)
  medium-1~3.txt    : ~50 KB 각   (중형 3개)
  large-1~2.txt     : ~1 MB 각    (대형 2개)

[테스트 A — 전체 한 번에 업로드]
루트 폴더에서 위 7개 파일 + mixed-batch/ 폴더 전체 선택
→ 업로드 시작
주의: world-map-meta.txt (5MB)는 Multipart 임계값(8MB) 미만이므로
      단일 PUT으로 처리됨

1. 업로드 진행률 표시 확인
   - 소형 파일들: 빠르게 완료
   - world-map-meta.txt: 상대적으로 오래 걸림
2. LogPanel에서 각 파일 별도 진행률 표시 확인
3. 모든 파일 업로드 완료 후 [전체 Purge]

[테스트 B — Multipart 파일 포함 혼합 업로드]
로컬에서 10MB 파일 추가 생성:
  PowerShell: [System.IO.File]::WriteAllBytes('bigfile-10mb.txt', [byte[]]::new(10*1024*1024))

1. 위 7개 + bigfile-10mb.txt 함께 업로드
2. bigfile-10mb.txt만 Multipart 처리 확인 (ETag에 -숫자 포함 여부)
3. Multipart 파일 업로드 완료 후 Purge 정상 동작 확인

[테스트 C — Smart Sync: 일부만 변경 후 재업로드]
1. 전체 업로드 완료 후
2. version.json.txt 내용만 수정:
   version: 1.2.4 (변경)
3. 전체 선택 업로드 → Smart Sync 결과 확인:
   - version.json.txt  → toOverwrite (변경됨) → 자동 Purge
   - 나머지 6개 파일   → toSkip (MD5 동일)
4. LogPanel에서 Skip 항목 확인

[테스트 D — 혼합 배치 Purge 성능]
mixed-batch/ (총 10개, 크기 혼합) 전체 Purge
- tiny 5개 + medium 3개 + large 2개 = 10개 경로
- Purge API는 크기 무관하게 경로만 사용하므로 동일 속도 예상

[기대 결과]
- 크기와 무관하게 모든 파일 업로드 성공 ✓
- 10MB 이상은 자동 Multipart 처리 ✓
- Smart Sync가 크기 무관하게 MD5로 정확히 비교 ✓
- Purge는 파일 크기 무관하게 경로 기준 처리 ✓
