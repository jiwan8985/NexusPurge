시나리오 19 — 전체 버킷 루트 Purge ("핵폭탄" 시나리오)
================================================================

⚠️  경고: 이 시나리오는 CDN 전체 캐시를 무효화합니다.
         실제 서비스 트래픽이 있는 환경에서는 오리진(S3)으로
         트래픽이 집중되어 과부하가 발생할 수 있습니다.
         테스트 환경(sklb-test)에서만 실행하세요.

목적: 원격 패널 경로를 버킷 루트(/)에 놓고 전체 Purge 실행 시
      앱이 어떻게 처리하는지 확인
      운영자가 실수로 루트 경로에서 "전체 Purge"를 누르는 상황

[순서]
1. 원격 패널을 버킷 루트 경로(/)로 이동
2. [전체 Purge] 버튼 클릭
3. PurgeDialog 경고 확인

[기대 동작]
Option A (추천): 루트 경로 전체 Purge 시 별도 경고 표시
   "현재 경로가 루트(/)입니다. 전체 CDN 캐시를 무효화합니다.
    계속 진행하시겠습니까?"

Option B (현재): 일반 전체 Purge와 동일하게 처리

[검증 포인트]
- contents/ 하위뿐만 아니라 다른 경로 캐시도 무효화되는지
- CloudFront /* Invalidation 정상 처리 여부
- Akamai CP_CODE 전체 Purge 처리 여부

[실제 발생 가능성]
게임사에서 "캐시 다 날려버려" 라는 지시를 받고
루트에서 전체 Purge 누르는 상황 → 매우 흔함

[권장 개선 사항]
현재 경로가 루트(/) 또는 최상위 prefix인 경우
추가적인 확인 팝업을 표시하는 기능 필요 (feature-candidates 11번 항목 관련)
