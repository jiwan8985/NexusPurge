================================================================
시나리오 26 — 다중 파일 개수별 업로드 + Purge
================================================================

목적: 파일 개수가 증가함에 따라 경고 임계값 동작과
      병렬 처리 성능을 검증

[폴더 구성]
count-010/  : 10개   (정상 범위)
count-050/  : 50개   (정상 범위)
count-100/  : 100개  (정상 범위)
count-500/  : 500개  (정상 범위)
count-1000/ : 1,000개 (경고 기준 근접 — 기본 경고: 5,000개)

[5,000 / 10,000개는 로컬 생성 필요]
PowerShell로 직접 생성:
  1..5000 | ForEach-Object { Set-Content "file-$($_.ToString('D5')).txt" -Value "idx $_" }

[테스트 A — 개수별 업로드 성능 측정]
각 폴더를 개별 업로드하면서 소요 시간 기록:
  10개   → 목표: 5초 이내
  50개   → 목표: 15초 이내
  100개  → 목표: 30초 이내
  500개  → 목표: 2분 이내
  1,000개→ 목표: 5분 이내

[테스트 B — 파일 수 경고 임계값 동작]
설정 → 파일 수 경고 기준을 80으로 낮춤
1. count-100 폴더 전체 선택 (100개)
2. 업로드 시도 → 파일 수 경고 팝업 표시 확인
   "선택한 파일이 100개입니다 (80개 이상).
    배치 처리 시간이 오래 걸릴 수 있습니다. 계속할까요?"

[테스트 C — 파일 수 제한 임계값 동작]
설정 → 파일 수 제한 기준을 200으로 낮춤
1. count-500 폴더 전체 선택 (500개)
2. 업로드 시도 → 강한 경고 팝업 표시 확인
   "S3/CDN 제한으로 인해 처리 시간이 매우 오래 걸리거나 실패할 수 있습니다."

[테스트 D — 1,000개 업로드 후 전체 Purge]
1. count-1000 폴더 업로드
2. [전체 Purge] 실행
3. 1,000개 경로 Purge 처리 시간 측정
4. Purge 배치 크기(기본 1,000개) 이내인지 확인

[테스트 E — 동시 전송 수 변경 성능 비교]
설정 → 동시 전송 수를 각각 1 / 4(기본) / 8 / 16으로 변경
count-500 폴더 업로드 소요 시간 비교:
  동시 1개: 가장 느림
  동시 4개: 기본값
  동시 16개: 가장 빠름 (네트워크 한계 도달 시 차이 미미)

[기대 결과]
- 개수 증가 시 경고 팝업 정상 트리거 ✓
- 동시 전송 수 증가 시 업로드 속도 향상 ✓
- 1,000개 Purge도 배치 1회로 처리 ✓
