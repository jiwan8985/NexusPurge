================================================================
시나리오 14 — 게임 긴급 핫픽스 배포 시뮬레이션
================================================================

목적: 게임 서비스 중 버그 발생으로 여러 데이터 파일을 동시에
      교체하고 즉시 CDN Purge가 필요한 긴급 상황 시뮬레이션

[배경]
게임사에서 실제로 발생하는 상황:
- 점검 없이 서버 사이드 데이터만 교체
- 클라이언트가 CDN에서 캐시된 이전 데이터를 읽고 있는 상태
- Purge 지연 시 잘못된 드롭률/가격/밸런스 데이터가 클라이언트에 노출

[핫픽스 대상 파일]
client_config.txt    : 클라이언트 기본 설정
event_banner_*.txt   : 이벤트 배너 (노출 중 교체)
shop_data.txt        : 상점 가격 데이터 (결제 오류 직결)
skill_balance.txt    : 스킬 밸런스 (PvP 불균형)
monster_table.txt    : 몬스터 스탯 (전투 불능)
drop_rate_table.txt  : 드롭률 (경제 붕괴 가능)
server_notice.txt    : 서버 공지
patch_manifest.txt   : 패치 매니페스트
version_info.txt     : 버전 정보

[순서]
1. *_BEFORE 파일 10개를 각각 원래 파일명으로 업로드
   (PATCHED_ 접두어 없이: client_config.txt 등)
2. CDN에서 모든 파일 캐시 확인 (Version: BEFORE_PATCH)
3. 핫픽스 패치 파일(PATCHED_*.txt)을 원래 파일명으로 교체 업로드
4. Smart Sync → 전체 10개 Overwrite 분류
5. 자동 Purge ON 상태에서 업로드 → 즉시 전체 Purge 실행
6. CDN 갱신 확인 (Version: AFTER_PATCH)

[타이밍 측정]
업로드 시작 ~ CDN Purge 완료 전체 소요 시간 측정
- Akamai: 목표 60초 이내
- CloudFront: 목표 5분 이내 (Invalidation 특성상)

[기대 결과]
- 10개 파일 모두 Overwrite 분류
- Purge 10개 경로 일괄 처리 (1회 API 호출)
- LogPanel: 업로드 완료 + CDN Purge 완료 전부 기록
- CDN: AFTER_PATCH 내용으로 즉시 갱신

[실패 시 확인]
- 일부 파일만 Overwrite 분류된 경우 → 파일 크기 동일 여부 확인
- Purge 지연 시 → CloudFront Invalidation 상태 LogPanel에서 추적
