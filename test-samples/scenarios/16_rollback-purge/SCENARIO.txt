시나리오 16 — 긴급 롤백 + Purge
================================================================

목적: 잘못된 파일 배포 후 이전 버전으로 즉시 롤백하고 CDN을 Purge하는 상황
      게임사에서 패치 후 치명적 버그 발견 → 구버전으로 되돌리는 긴급 대응

[배경]
2.0.0 배포 후 아이템 드롭률 0% 버그 발견
→ 1.0.0 stable 버전으로 즉시 롤백 필요
→ CDN에 2.0.0 캐시가 남아있으면 유저가 계속 버그 버전을 읽음

[순서]
1. stable-v1.txt 를 data.txt 이름으로 업로드 (정상 운영 상태)
2. CDN 확인: Status: STABLE

3. broken-v2.txt 를 data.txt 이름으로 교체 업로드 (잘못된 패치)
4. 자동 Purge 실행 → CDN에 broken 버전 반영
5. CDN 확인: Status: BROKEN (버그 배포된 상태)

6. [롤백] stable-v1.txt 를 data.txt 이름으로 재업로드
7. Purge 즉시 실행 → CDN에서 broken 버전 제거
8. CDN 확인: Status: STABLE (롤백 완료)

[시간 측정]
버그 배포 인지 → 롤백 완료까지 전체 소요 시간 기록
목표: Akamai 3분 이내 / CloudFront 10분 이내

[기대 결과]
- 롤백 후 CDN 즉시 갱신: Status: STABLE 표시
- LogPanel에 전체 흐름 기록 (v1→v2→v1 복원)
- 유저 측면: broken 버전 캐시 시간 최소화

[실패 시 대응]
Purge가 지연되는 경우:
→ CDN URL에 ?v=rollback 쿼리스트링으로 임시 우회 (캐시 미스 유도)
→ CloudFront Invalidation 상태 확인 (LogPanel Invalidation ID 추적)
