시나리오 08 — 버전 사이클 Purge (릴리즈 배포 패턴)
================================================================

목적: 실제 서비스 배포 시 버전 파일만 교체하고 Purge하는 흐름 검증

[파일 구성]
release-v1.txt   : 1.0.0 릴리즈 노트 (최초 배포)
release-v2.txt   : 2.0.0 릴리즈 노트 (업데이트)
release-v3.txt   : 3.0.0 릴리즈 노트 (최신)
static-resource  : 변경 없는 정적 파일 (Purge 불필요)

[배포 흐름 시뮬레이션]

1단계 — 초기 배포
  release-v1.txt 를 release.txt 이름으로 업로드
  CDN 캐시 확인: "Version: 1.0.0" 표시

2단계 — v2 배포
  release-v2.txt 를 release.txt 이름으로 교체 업로드
  자동 Purge 트리거 → CDN 갱신: "Version: 2.0.0"

3단계 — v3 배포
  release-v3.txt 를 release.txt 이름으로 교체 업로드
  자동 Purge 트리거 → CDN 갱신: "Version: 3.0.0"

[static-resource 검증]
- static-resource.txt 는 단 한 번 업로드 후 수정 없음
- 매 배포 시 Smart Sync가 Skip으로 분류하는지 확인
- Purge 대상에 포함되지 않는지 확인

[기대 결과]
- release.txt: 배포마다 CDN 내용 갱신 ✓
- static-resource.txt: Purge 대상 제외 ✓
- 각 Purge 후 X-Cache: Miss → 재캐시 후 Hit ✓
