## 1. 프로젝트 개요

본 프로젝트는 파편화된 CDN 콘텐츠 관리 프로세스를 통합하고, S3 업로드와 CDN Purge(무효화) 과정을 자동 연동하여 운영 효율성을 극대화하는 것을 목적으로 합니다. 특히 전문 기술 지식이 부족한 일반 운영자도 쉽게 사용할 수 있도록 **사용자 편의성(UX)** 강화에 중점을 둡니다.

## 2. 요구사항 정의 (Requirements)

### 2.1 핵심 기능 (Must-Have)

- **멀티 스토리지 업로드**: AWS S3 및 Tencent Cloud COS 등 S3 호환 API를 사용하는 모든 객체 스토리지 지원.
- **지능형 파일 동기화 (Smart Sync)**: 전체 업로드가 아닌, 로컬과 원격지의 MD5 해시값 비교를 통해 변경된 파일만 식별하여 업로드.
- **조건부 자동 Purge**: 파일이 덮어쓰기(Overwrite)된 경우에만 해당 콘텐츠의 CDN 캐시 URL을 즉시 무효화.
- **멀티 CDN 어댑터**: AWS CloudFront, Tencent Cloud CDN 등 다양한 벤더의 API를 단일 인터페이스로 제어.
- **사용자 친화적 인터페이스 (GUI)**: 복잡한 명령어가 아닌 드래그 앤 드롭 및 클릭 기반의 직관적인 UI 제공.

### 2.2 부가 기능 (Nice-to-Have)

- **실시간 진행률 표시**: 업로드 및 Purge 진행 상태를 프로그레스 바(Progress Bar)로 시각화.
- **작업 이력 로깅**: 업로드 및 Purge 성공/실패 여부를 로그 파일 및 UI 상의 히스토리로 기록.
- **Dry-run 모드**: 실제 반영 전 변경될 파일 리스트 및 Purge 대상 경로 미리보기 기능.
- **병렬 처리**: 다량의 파일 전송 시 멀티쓰레딩/비동기 처리를 통한 속도 최적화.
